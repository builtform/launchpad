# LaunchPad v1.1.0

**Minor release: enforces evidence-before-claims for every completion assertion.**

A new harness skill closes the most common agentic failure mode where work is declared "done" without actually running the checks. When an agent is about to claim tests pass, the build is green, a PR is ready, or any other completion-shaped assertion, the skill auto-triggers and refuses the claim until fresh verification command output is attached. No new commands; no breaking changes; install path unchanged.

## Added

- **`lp-verification-before-completion` skill** at `plugins/launchpad/skills/lp-verification-before-completion/`. Auto-triggers on completion-claim phrasing across commands ("tests pass", "build green", "PR ready", "Definition of Done met", "fixed", "done") and refuses claims that lack attached command output. Maps each kind of claim to the verification command that proves it (test, typecheck, lint, build) and requires the command's exit code and output to be visible in the same message as the claim. Closes the most common agentic failure mode where work is declared "done" without running the checks. Adapted from [obra/superpowers](https://github.com/obra/superpowers) (MIT). Includes 4 eval scenarios covering happy path, partial-verification temptation, negative boundary (no-op), and sub-agent delegation trust prevention.

## Stack handling

The TS-stack pnpm examples in the skill prose are illustrative. For other stacks, the verification commands resolve to whatever `.launchpad/config.yml` declares under `commands.test`, `commands.typecheck`, `commands.lint`, and `commands.build`, dispatched via `${CLAUDE_PLUGIN_ROOT}/scripts/plugin-build-runner.py --stage=<stage>`. The skill's contract is the same regardless of stack: identify the right stage, run it fresh, attach the output.
