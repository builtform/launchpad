# LaunchPad v2.1.1

## v2.1.1 ship (this commit, post-PR-#62)

**v2.1.1 dogfoods its own quality gates.** The mandatory `/lp-review` dual-pass at `/lp-commit` and `/lp-ship`, the `--no-context` blind-review flag, and the Layer 3 static-analysis stack (ruff + bandit + semgrep + pyright + pytest) all landed via this very release — Phases 0-5 plus three iterations of cross-cutting sweep-review were the first production exercise of the full Layer 1 + Layer 2 + Layer 3 stack.

User-visible deltas over v2.1.0:

- **Mandatory dual-pass review at `/lp-commit` Step 2.5 + `/lp-ship` Step 4.6.** Was previously an optional yes/no prompt. A specialist pass (full project context) and a blind pass (`--no-context`) now run sequentially on every commit. Bypass is gated to emergency-hotfix paths only: `--skip-review` is honored just on hotfix branches with a TTY-guarded `BYPASS REVIEW` confirmation, and emits a trailing audit-trail trailer in the commit message so the bypass remains visible in `git log` indefinitely. **Behavior change for users.**
- **`/lp-ship` 3-round autonomous fix loop with 90-min timeout.** Step 4.6 now runs up to three rounds of dual-pass-then-resolve, gating the resolver on a sensitive-file denylist (auth / middleware / security / migrations / CI config) and a per-finding staged-diff scope check (`origin/main..HEAD`) so the auto-fix does not stray outside the PR's actual diff range.
- **`/lp-review --no-context` flag.** Bias-stripped blind pass that runs the same review agents without the broader project context surface — catches what context can mask. Used internally by Step 2.5's blind pass; available standalone for users who want a second opinion on a diff without their familiarity coloring the lens.
- **New guide: [`docs/guides/CODE_REVIEW_LAYERS.md`](../guides/CODE_REVIEW_LAYERS.md).** The three-layer review architecture (Layer 1 commit-message + lint, Layer 2 dual-pass review, Layer 3 static-analysis gates) is documented end-to-end with override patterns for each layer.
- **`CLAUDE.md` Definition of Done extended with conditional Python gates.** Apply only when changes touch `*.py`. TypeScript / JavaScript DoD is unchanged.

The Universal `lefthook.yml` self-host hardening (routing test/typecheck/lint through `plugin-build-runner.py` for stack-aware dispatch) ships at v2.1.1 **for plugin maintainers only**. Downstream lefthook propagation — adding bandit/ruff/pyright/pytest to consumer projects' pre-commit and pre-push gates via stack-adapter fragments — is tracked as BL-316 and scheduled for v2.1.2.

## Highlights

- **Phase 1: `--no-context` flag on `/lp-review`.** Adds a context-stripping invocation mode so the same dispatched agents see only the diff, not broader repo memory. Used by Step 2.5's blind pass; available standalone via `/lp-review --no-context`.

- **Phase 2: Mandatory dual-pass at `/lp-commit` + `/lp-ship`.** Two `/lp-review` passes (specialist with full context, then blind) become a hard contract on every commit. Bypass requires explicit `--skip-review` with TTY guard + audit-trail trailer.

- **Phase 3: Universal `lefthook.yml` + `plugin-build-runner.py` indirection.** `test`/`typecheck`/`lint` route through a single Python orchestrator that reads `.launchpad/config.yml` `commands.*` arrays for stack-aware dispatch. Self-host only at v2.1.1; downstream propagation via stack-adapter fragments is BL-316 / v2.1.2.

- **Phase 4: Layer 3 static-analysis stack on plugin scripts.** ruff + bandit + semgrep (general + LaunchPad-internal) + pyright + pytest wired to lefthook for the plugin's own Python tree. pip-compile-locked `requirements.txt` with `--require-hashes` provides supply-chain coverage. Pyright strict on three security-boundary modules (`decision_validator.py`, `nonce_ledger.py`, `decision_integrity.py`) with documented `# type: ignore` ladders for residual `Any`-leakage at subprocess boundaries.

- **Phase 5: Verification + ship discipline.** Phase 5 shipped the docs (CODE_REVIEW_LAYERS.md, CI_CD.md inventory, CHANGELOG `## [v2.1.1]` block) and proved the new gates by being the first commit run through them. Three sweep-review iterations converged clean by iteration 3 — 84 unique `tool_use_ids` captured across the ship lane, full Hard Rule 6 compliance throughout.

## Migration from v2.1.0

No data migration required. v2.1.1 is purely additive review/lint surface. Existing v2.1.0 scaffolds, sealed identity contracts, decision schemas, and harness state continue to work unchanged.

The behavior change to be aware of: any project using `/lp-commit` or `/lp-ship` will now invoke the dual-pass review automatically on every commit. If you previously declined the optional review prompt, it now runs by default. To bypass on a one-off hotfix, use `--skip-review` on a hotfix branch with TTY confirmation — the bypass writes an audit-trail trailer to the commit message.

### Known limitations

- **Downstream lefthook propagation not yet shipped.** Consumer projects installing the plugin do not automatically get the new bandit/ruff/semgrep/pyright/pytest gates in their `lefthook.yml` — only the plugin's own self-host config has them. Consumer-side propagation via stack-adapter fragments is tracked as BL-316 and scheduled for v2.1.2.
- **Layer 3 macOS `timeout` wrapper removed.** The `timeout` binary is absent on macOS by default; portable wrapper deferred to BL-305 (v2.1.3). Linux lefthook entries continue to honor timeouts.
- **Pyright strict not extended to engine modules.** `lp_pick_stack/engine.py` (23 errors) and `lp_scaffold_stack/engine.py` (108 errors) remain at standard mode; per-module strict deferred to BL-298 (v2.1.3) since engine modules are subprocess/path-manipulation surfaces with lower bang-for-buck than security-boundary modules.

## Security

- **Bandit `-ll` (medium-or-higher severity) on plugin scripts.** Wired to lefthook pre-commit. `LOW`-severity findings are suppressed at v2.1.1; tracked for re-evaluation at v2.1.x.
- **Semgrep universal + LaunchPad-internal rule packs.** General pack covers eight cross-language patterns (path traversal, command injection, weak random, etc.). LaunchPad-internal pack enforces three architectural invariants; the fourth invariant (sentinel-precedes-materialize) is deferred to BL-303 per >2 fail-to-validate iterations during Phase 4.
- **Pyright strict on the 3 security-boundary modules.** `decision_validator.py`, `nonce_ledger.py`, `decision_integrity.py` carry strict mode + documented `# type: ignore` ladders for residual `Any`-leakage at subprocess boundaries.
- **`nodeenv` supply-chain caveat.** Pyright transitively pulls `nodeenv`, which downloads a Node.js binary at install time outside pip's hash protection. Tracked as BL-299 for v2.1.3 remediation. Operators in regulated environments should pin/verify the node binary out-of-band.
- **Tag-signing automation deferred.** v2.1.1 tag is unsigned per the existing posture; signing automation lands at v2.2 (BL-258). See SECURITY.md.

## Upgrading from v2.1.0

```
/plugin marketplace update builtform
/plugin uninstall launchpad --scope <same-scope>
/plugin install launchpad@builtform --scope <same-scope>
```

Restart Claude Code after updating. No migration step is required — v2.1.0 → v2.1.1 is purely additive review/lint surface; existing project state is preserved unchanged.
