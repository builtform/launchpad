# LaunchPad v2.1.9

## v2.1.9 ship: Dependency sweep + v2.2 planning seed

**v2.1.9 is a maintenance release.** No plugin code changes; no behavior delta for downstream consumers. The release captures ten post-v2.1.8 maintenance commits: nine Dependabot dependency bumps (five patches and four majors) and one v2.2 backlog seed (BL-374, TypeScript 5.x to 6.x upgrade audit).

User-visible deltas over v2.1.8: none. The dependency bumps update build-time and dev-time toolchain only; the backlog seed is documentation. Marketplace consumers see the version pin advance from `2.1.8` to `2.1.9` but no functional change.

---

## Highlights

- **Dependency sweep landed in two waves.** Five patch-level bumps (`hono`, `turbo`, `typescript-eslint`, `semgrep`, `pnpm/action-setup`) plus four major-version bumps (`pytest`, `@eslint/js`, `actions/setup-python`, `osv-scanner-action`). Every bump verified CI-green on its pre-merge stale base; the majors are toolchain-only and backward-compatible at the consumed-surface level. The bumps landed via squash-merge with admin override after the v2.1.8 release was tagged, then the version pin caught up here.
- **TypeScript 6.x deferred to v2.2 (BL-374).** Dependabot also proposed `typescript 5.9.3 -> 6.0.3` (PR #74), but it failed Type Check + Build with six errors in `apps/api/src/index.ts` (missing `process`, `console`, `NodeJS` namespace). Root cause is TS 6.0's tighter type auto-loading: the shared `packages/typescript-config/node.json` needs an explicit `"types": ["node"]` for any Node-target package to compile. The one-line fix is known, but TS major bumps historically expose multiple breaking-change surfaces (control-flow inference, lib.d.ts moves, module-resolution refinements), so PR #74 was closed and the work is captured as BL-374 for a deliberate single-PR audit in v2.2.

## Backlog additions

### BL-374 (v2.2, P2): TypeScript 5.x to 6.x upgrade audit

Filed from the PR #74 review on 2026-05-17 (closed without merging). TypeScript 6.0 stopped auto-loading `@types/*` packages from `node_modules`; the shared `packages/typescript-config/node.json` needs an explicit `"types": ["node"]` declaration for `apps/api` and `packages/db` to compile. The one-line fix is known, but the wider audit covers control-flow inference tightening, `lib.d.ts` moves, and module-resolution refinements that TS 6.0 introduced.

Deferred to a single-PR audit in v2.2 rather than a Dependabot-driven auto-merge. See `docs/tasks/BACKLOG.md` (BL-374 entry) for the full audit scope, files-to-modify list, acceptance criteria, and stop conditions.
