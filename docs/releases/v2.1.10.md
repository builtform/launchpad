# LaunchPad v2.1.10

## v2.1.10 ship: Freshness gate no longer blocks unrelated PRs + dependency sweep

**v2.1.10 is a maintenance release with one plugin-internal CI fix.** There is no behavior delta for downstream consumers: the fix touches LaunchPad's own release/CI tooling, and the remaining commits are dependency bumps plus one backlog seed. Marketplace consumers see the version pin advance from `2.1.9` to `2.1.10` with no functional change.

The headline fix removes a self-inflicted CI failure mode: the `v2-handshake-lint` required check enforced a 30-day `last_validated:` freshness window on every pull request, so once 30 days elapsed since the last catalog re-stamp it began failing **every** PR (including unrelated Dependabot bumps), regardless of what the PR changed. v2.1.10 makes that staleness advisory on PRs and enforces the window where it belongs: at release time.

---

## Highlights

- **Catalog freshness is now advisory on PRs and hard-gated at release (#104).** The freshness check is a time-based maintenance cadence with no relationship to the content of any given PR, so it no longer blocks unrelated changes. Structural problems (missing / malformed / future-dated `last_validated:`) remain hard failures; only the lapsed-window case is downgraded to a non-blocking advisory. The 30-day window is enforced at tag time by two release-workflow gates: a catalog/pattern integrity gate (`plugin-v2-handshake-lint.py --check-freshness`) and the full OPERATIONS §4 contract gate (`plugin-freshness-check.py --gating`). A new `RELEASE_PROCESS.md` step documents re-validating and re-stamping the §4 docs before tagging so the release gate is an expected checklist item rather than a surprise. Severity classification was also hardened to branch on intrinsic control flow rather than substring-matching an error message, closing a malformed-date misclassification path.
- **Dependency sweep.** Seven Dependabot bumps (one major GitHub Action, the rest patch/minor) plus one transitive security bump, all toolchain- or template-scoped.

## For LaunchPad users (downstream behavior changes)

None. v2.1.10 is a LaunchPad-repo CI fix plus build-time and dev-time toolchain maintenance. Projects scaffolded from LaunchPad do not run the `v2-handshake-lint` freshness gate (it is LaunchPad's own workflow), so the headline fix has no downstream effect.
