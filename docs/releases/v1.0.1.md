# LaunchPad v1.0.1

**Patch release: dependency hygiene and a second AI code reviewer for every PR.**

23 of 26 CVEs flagged on the public-flip scan are cleared via a recursive dependency sweep, and Greptile joins Codex as a complementary cross-file / codebase-aware AI reviewer on every pull request. No production code changes — the plugin behavior at install is identical to v1.0.0; this release is repo hygiene plus tooling.

## Security

- **23 of 26 flagged CVEs cleared** via direct dep bumps + `pnpm update --recursive && pnpm dedupe`:
  - High-severity fixes: `hono` → 4.12.15 (arbitrary file access via serveStatic), `@hono/node-server` → 1.19.14 (auth bypass via encoded slashes), `next` → 15.5.15 (Server Components DoS), `picomatch` (ReDoS), `defu` (proto pollution), `effect` (concurrency context contamination), `flatted` (proto pollution)
  - Medium-severity fixes: cleared 12+ medium CVEs across `hono`, `next`, `picomatch`, `brace-expansion`, `postcss`, `vite`, and others
- **3 residual CVEs** in deeply-transitive copies (esbuild 0.21.5, postcss 8.4.31, vite 5.4.21) pinned by other dependencies; addressable via `pnpm.overrides` in v1.0.2 if upstream Dependabot keeps flagging them
- **GitHub Actions runners upgraded** to current major versions (commit-pinned): `actions/checkout` v6.0.2, `actions/setup-node` v6.4.0, `actions/cache` v5.0.5

## Added

- **Greptile as a second AI code reviewer** alongside Codex on every PR:
  - Codex covers the **narrow / line-level** lane (per-PR diff context)
  - Greptile covers the **wide / codebase-aware** lane (pre-indexed graph of the whole repo)
  - Both advisory only; merge gating remains the existing required `ci.yml` jobs
  - Configured via `greptile.json` at repo root (strictness=1, advisory-only, dependabot[bot] excluded)
  - LaunchPad qualifies for Greptile's free OSS Program (MIT license)
- **Template support for downstream projects**: `greptile.template.json` + `init-project.sh` swap-chain wiring so projects scaffolded from the LaunchPad template inherit the dual-reviewer pattern automatically. `{{PROJECT_NAME}}` placeholder in the template substitutes into the per-project `instructions` field at init time.

## Improved

- `/lp-commit`, `/lp-ship`, and `/lp-build` PR-monitoring loops now include Gate B3 for Greptile alongside Gate B2 for Codex
- Gate numbering aligned across `/lp-commit` and `/lp-ship` (B1 = human review, B2 = Codex, B3 = Greptile) so both flows can be reasoned about with one mental model
- `/lp-ship` autonomous auto-fix criteria for Greptile findings now use concrete signals (P0/P1 severity + cross-file evidence) rather than ambiguous "high-confidence" wording

## Documentation

- New `docs/architecture/CI_CD.md` — full dual-reviewer reference, Dependabot hygiene, merge protocol
- `docs/guides/HOW_IT_WORKS.md` — tech-stack table updated; new "Setting up Greptile" subsection covering per-project install steps that cannot be automated (App install, OSS application, dashboard indexing)
- `SECURITY.md` — new hardening recommendation describing the dual-reviewer pattern with proper narrow vs. wide framing
- `docs/architecture/REPOSITORY_STRUCTURE.md` + `scripts/maintenance/check-repo-structure.sh` — root-file whitelist now allows `greptile.json` and `greptile.template.json`

## Internal

- New CI workflow `.github/workflows/release-notes-check.yml` (LaunchPad-only, removed from downstream by `init-project.sh`) — enforces that every release PR and tag push includes a hand-authored `docs/releases/v<VERSION>.md` file. Prevents the oversight where a release ships with auto-generated diff notes instead of curated release notes.
- New maintainer-only doc `docs/maintainers/RELEASE_PROCESS.md` (LaunchPad-only) — explicit step-by-step release checklist with the `release-notes-file-first` rule called out as step 1.
- All 15 plugin tests pass; full `ci.yml` required-checks suite (Type Check, Lint, Build, Test, Repo Structure, Install) green.

## Upgrading from v1.0.0

```
/plugin marketplace update builtform
/plugin uninstall launchpad --scope <same-scope>
/plugin install launchpad@builtform --scope <same-scope>
```

Restart Claude Code after updating.
