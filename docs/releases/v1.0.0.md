# LaunchPad v1.0.0

**Installable Claude Code plugin for the LaunchPad autonomous-engineering harness — runs end-to-end in any brownfield repo, no template clone required.**

Published under the **BuiltForm** marketplace. Ships 38 slash commands, 36 sub-agents, and 16 skills, wired through a brownfield-safe adaptation pipeline (`/lp-kickoff` → `/lp-define` → `/lp-plan` → `/lp-build`) across TypeScript, Python, Django, Go, and polyglot codebases.

## Highlights

- **Plugin-first architecture.** Install with one command in any repo:

  ```
  /plugin install launchpad@builtform
  ```

  After a Claude Code restart, every `/lp-*` command is available — no template clone, no file copying, no repo surgery.

- **Brownfield-safe `/lp-define`.** Detects the project's actual stack from `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `Gemfile`, and `composer.json`, then scaffolds adapted canonical docs (`PRD.md`, `TECH_STACK.md`, `BACKEND_STRUCTURE.md`, `APP_FLOW.md`), a `SECTION_REGISTRY.md`, `.launchpad/config.yml`, and `.launchpad/agents.yml` — all shaped to what was detected. Polyglot projects (TS + Python together) get correctly-merged configs and agent rosters out of the box.

- **Config-driven L2 commands.** `/lp-commit`, `/lp-ship`, `/lp-review`, `/lp-test-browser`, `/lp-harden-plan` read shell invocations from `.launchpad/config.yml` instead of hardcoded `pnpm` commands. The same command works in a TypeScript monorepo, a Django backend, a Go CLI, or a polyglot repo — differences live in config, not spec.

- **Security-hardened doc generator.** Secret scanner runs against every generated doc (refuses write on AWS, GitHub, Stripe, Anthropic, OpenAI, Slack, and DB-credential patterns). Manifest stripper redacts `scripts.*`, registry tokens, and URLs with embedded credentials across six package-manager manifests. Jinja2 `autoescape=True` on every template prevents server-side template injection via hostile package.json fields. Realpath-confined writes reject any `paths.*` value that escapes the repo root.

- **Audit trail built in.** Every autonomous `/lp-build`, `/lp-ship`, `/lp-commit` invocation is recorded in `.launchpad/audit.log` with a content-hash of the canonical `commands` config section that survives rebase/amend/squash. `LP_CONFIG_REVIEWED` env-var pin lets CI opt into autonomous runs without blanket "allow any change" escape hatches.

## Installation

```bash
# One-time, from any repo where you want /lp-* commands:
/plugin marketplace add https://github.com/builtform/launchpad
/plugin install launchpad@builtform

# Then restart Claude Code.
```

Local-path install (for contributors or private forks) works the same way — replace the URL with an absolute filesystem path:

```bash
/plugin marketplace add /absolute/path/to/launchpad-clone
/plugin install launchpad@builtform
```

The marketplace is named **`builtform`**; the plugin is **`launchpad`** — hence the `launchpad@builtform` install syntax.

> **Path-with-spaces gotcha:** if your local path contains spaces, invoke `/plugin marketplace add` with no args to use the interactive dialog. Argv parsing on the command-line form does not quote-escape correctly. Documented v1.1 polish item.

## What's New

### Plugin packaging

- **`.claude-plugin/` manifests.** Repository ships `.claude-plugin/marketplace.json` (name: `builtform`, schema-valid) and `plugins/launchpad/.claude-plugin/plugin.json` (name: `launchpad`, version: `1.0.0`, MIT-licensed). `claude plugin validate .` passes cleanly.
- **Canonical plugin layout.** `commands/`, `agents/`, and `skills/` live at the plugin root (`plugins/launchpad/`), matching every reference Claude Code plugin (Anthropic's `compound-engineering`, `frontend-design`, `agent-sdk-dev`).
- **Plugin runtime scripts in-tree.** All 7 Python/shell scripts that power harness commands (`plugin-stack-detector.py`, `plugin-doc-generator.py`, `plugin-config-loader.py`, `plugin-build-runner.py`, `plugin-audit-log.py`, `plugin-config-hash.py`, `plugin-prereq-check.sh`) live inside `plugins/launchpad/scripts/` and resolve via `${CLAUDE_PLUGIN_ROOT}`, so they're present in the install cache.
- **Auto-install from `init-project.sh`.** Greenfield users scaffolding a fresh LaunchPad project get the plugin registered + installed automatically, with graceful degradation if the `claude` CLI is missing.

### Brownfield adaptation

- **Stack detector.** Allowlist-only manifest reader — never touches `.env*`, `.npmrc`, `secrets.yml`. Bounded walk (root + 1 level), hard-excludes `node_modules`, `.venv`, `__pycache__`, `dist`, `.next`, `target`, `build`, `vendor`, `.git`. Size cap (1 MB/manifest). Reports `{stacks, frameworks, polyglot, manifests, zero_manifest}` deterministically — alphabetically sorted, same input produces the same output.
- **Stack-adapter package.** Concrete adapters for `ts_monorepo`, `python_django`, `go_cli`, `generic`, plus a `polyglot` composer that merges multi-stack outputs deterministically (TS > Python > Go > generic precedence).
- **Vendored pure-Python runtime.** Jinja2 + MarkupSafe + PyYAML bundled in `plugins/launchpad/scripts/plugin_stack_adapters/_vendor/`. Zero `pip install` at plugin-install time; works on any Python 3.10+.
- **Config loader.** Single `.launchpad/config.yml` with three sections (`pipeline`, `commands`, `paths`). Section-by-section re-parse — a bad `commands:` block does not block `/lp-kickoff`. Realpath-confinement on every `paths.*` value. `commands.*` always-array schema (`test: []` = skip; `test: ["pnpm test", "pytest"]` = polyglot serial).
- **Single shared Step 0 helper.** `plugin-prereq-check.sh` with `--mode=lite` and `--mode=full` modes, session cache keyed on `config.yml` mtime + top-level manifests. Warm latency 20–30 ms. Structural enforcement that lite mode is a strict subset of full mode.

### Doc generator and canonical docs

- **`/lp-define` canonical-doc generator.** Detects → composes adapters → renders via Jinja2 → scans for secrets → writes. Six templates plus `agents.yml.j2`.
- **Section registry.** Lives in its own file at `docs/tasks/SECTION_REGISTRY.md` — clean separation from `PRD.md`. Downstream commands read the registry via a shared `section_registry.load_sections()` helper.
- **Stack-aware `agents.yml`.** Polyglot-conditional roster: TS detected → adds TS-specific reviewers; Python detected → adds Python reviewers; both → both. Re-running `/lp-define` never silently clobbers user-tuned edits.
- **Non-interactive safe.** No TTY and no `--force` ⇒ default keep. Canary-safe for CI.
- **Overwrite menu.** Interactive `[k]eep / [o]verwrite / [d]iff / [a]ll-overwrite / [s]kip-all` — `.launchpad/config.yml` and `.launchpad/agents.yml` excluded from `[a]ll-overwrite`.

### Guards and gating

- **Autonomous-build guard.** `.launchpad/autonomous-ack.md` committed-file requirement is the social signal. Refuses to run when section spec and ack file land in the same commit (the pattern a hostile PR would use).
- **Commands-hash CI override.** `LP_CONFIG_REVIEWED` env-var pin accepts the full 64-char SHA-256 or a 16-char prefix. Mismatch refuses with both expected and received hashes printed to stderr. `plugin-config-hash.py` is the canonical helper for regenerating the pin.
- **Pipeline-aware skip gates.** `/lp-test-browser` checks `pipeline.build.test_browser` and exits cleanly on `skipped` — backend-only projects bypass the command by configuration, not ad-hoc detection.

### Telemetry and audit

- **Audit log.** Gitignored by default (opt into committed mode via `audit.committed: true` in `config.yml`). Entries record ISO-8601 timestamp, git user, short HEAD sha, 16-char `commands` content-hash, and invoked command. Content-hash survives rebase/amend/squash.
- **Generator summary JSON.** `/lp-define` writes `{written, kept, skipped, gitignore_updated, dirs_created}` — meaningful telemetry for users inspecting what the command changed.

### Recommended companions

LaunchPad is intentionally narrow in scope. Two third-party tools pair well:

- **[Destructive Command Guard (dcg)](https://github.com/Dicklesworthstone/destructive_command_guard)** — pattern-based shell-command guard that closes the `--dangerously-skip-permissions` gap. **Strongly recommended** for any unattended `/lp-build` run. See [SECURITY.md](../../SECURITY.md).
- **[MemPalace](https://github.com/MemPalace/mempalace)** — local vector store + MCP server for verbatim session-memory recall. Setup cookbook in [MEMPALACE_INTEGRATION.md](../guides/MEMPALACE_INTEGRATION.md).

LaunchPad does not bundle either, does not auto-install them, and runs identically without them.

## Breaking changes vs v0.1

If you installed the pre-v1 plugin extraction (`v0.1.0`), you'll need to:

1. **Uninstall the old marketplace.** The marketplace name changed from `launchpad` to `builtform`:
   ```
   /plugin uninstall launchpad@launchpad
   /plugin marketplace remove launchpad
   ```
2. **Re-add under the new name + reinstall:**
   ```
   /plugin marketplace add https://github.com/builtform/launchpad
   /plugin install launchpad@builtform
   ```
3. **Plugin content moved.** `commands/`, `agents/`, `skills/` are now under `plugins/launchpad/` (not the repo root, not under `.claude/`). No action required unless you were sourcing files out of the repo by path.
4. **Install command string changed.** Any scripts referencing `/plugin install launchpad@launchpad` need to flip to `/plugin install launchpad@builtform`.

v0.1 was never merged to `main`. If you never installed it, this section does not apply.

## Known limitations (deferred to v1.1)

Carried forward into the v1.1 roadmap:

**Stack detection and generator polish:**

- Python framework detection treats any `pyproject.toml` as `python_django`. FastAPI / Flask projects need manual edit to `.launchpad/config.yml` until the framework-distinction adapter ships.
- Polyglot design-field precedence defaults to `skipped` for TS+Python combinations. Workaround: hand-edit `pipeline.define.design`, `pipeline.plan.design_review`, and `pipeline.build.test_browser` in `.launchpad/config.yml`. Composer precedence rule for design-related fields will explicitly favor TS `enabled` in v1.1.
- Bare `pytest` invocation assumes no env manager. Detection of `poetry.lock` / `uv.lock` is queued.

**Ergonomics:**

- Runner-fatal exit code `2` overlaps `pytest`'s collection-error exit. Remap to the reserved 64–78 range is queued.
- `/lp-pnf` and `/lp-brainstorm` Phase 1 are dialogue-heavy; no `--stub` / `--yes-to-all` non-interactive mode yet.
- `/lp-shape-section --from-todo <id>` import adapter for projects with existing TODO files is queued.

**Docs:**

- Install guide should explicitly document `scope: "local"` vs `scope: "user"` semantics.
- Install guide should recommend the no-args interactive `/plugin marketplace add` when paths contain spaces.
