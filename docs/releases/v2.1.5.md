# LaunchPad v2.1.5

## v2.1.5 ship — Tier 1 universal scope-review fixes

**v2.1.5 lands 15 universal fixes surfaced during a post-v2.1.4 real-world installed-plugin reproduction.** Pre-v2.1.5 every greenfield TS-stack first push failed CI red at the pnpm/action-setup or actions/setup-node step (missing version source / `.nvmrc`); brainstorm content captured by `/lp-brainstorm` was discarded when `/lp-define` rendered empty placeholder docs; CODEOWNERS shipped a literal `<copyright-holder>` placeholder under PII opt-out; the lefthook EOF-newline gate refused LaunchPad's own canonical envelopes; `pnpm astro check`-style commands false-passed the build runner on non-TTY stdin.

This is a Tier 1 release. Tier 2 stack-aware refactors (BL-345 through BL-352) ship as v2.1.6 in a separate PR per scope decision — the 11-stack matrix would have made a combined PR ~80 files, and reviewer signal-to-noise at that size is poor.

User-visible deltas over v2.1.4:

- **Greenfield TS-stack CI is green on first push (BL-353 + BL-354, both P0).** The rendered `.github/workflows/ci.yml` declares `version: '<DEFAULT_PNPM_VERSION>'` on `pnpm/action-setup`; `/lp-bootstrap` renders `.nvmrc` containing `<DEFAULT_NODE_VERSION>` so `actions/setup-node`'s `node-version-file:` resolves. Both versions live in `plugin_stack_adapters/_constants.py` — single source of truth. Pre-fix every greenfield TS-stack first push failed at setup; all downstream steps were skipped.
- **`/lp-bootstrap` refuses inconsistent workflow + file-reference state (BL-355, P1).** Structural assertion at render time: every rendered workflow's `*-version-file:` input MUST name a file in the bootstrap render batch or already on disk. Catches the BL-353/354-class regression at write time, before the user pushes.
- **`/lp-define` consumes `.launchpad/brainstorm-summary.md` into the canonical docs (BL-333, P0).** PRD/APP_FLOW/BACKEND_STRUCTURE consume `brainstorm.<section_slug>` content when present, wrapped in a verifiable `<!-- v2.1.5 BL-333: filled from .launchpad/brainstorm-summary.md — verify and edit. -->` comment block. Common brainstorm headings map to canonical section slugs via a small alias table (`Problem` → `overview`, `Personas` → `users`, `Goals` → `success_criteria`).
- **Public-repo prerequisites all render (BL-336 + BL-342 + BL-343 + BL-344).** `.github/dependabot.yml`, `.github/pull_request_template.md`, `docs/architecture/REPOSITORY_STRUCTURE.md` all ship via `/lp-bootstrap` + `/lp-define`. `SECURITY.md` was already in KernelRenderer's kernel-file list from a prior v2.1.x release — locked down with a regression test.
- **CODEOWNERS no longer ships a broken owner line under PII opt-out (BL-334).** PII-opt-out path emits a `# TODO: set primary owner via /lp-update-identity --copyright-holder <handle>` comment instead of routing review requests to a non-existent placeholder user.
- **lefthook EOF-newline excludes LaunchPad's canonical envelopes (BL-339).** The `end-of-file-newline` hook excludes `.launchpad/(scaffold-decision|scaffold-receipt)\.json$` so the hook doesn't reject LaunchPad's own byte-deterministic JSON envelopes.
- **`plugin-build-runner.py` no longer false-passes on non-TTY interactive prompts (BL-340).** Tees stdout/stderr to terminal AND scans for known prompt patterns (`Continue? Yes / No`, `[Y/n]`, `Do you want to install`). Exit 0 + prompt detected = failure with a clear error.
- **`/lp-review` works pre-first-commit (BL-337); `/lp-commit` accepts initial-scaffold commits on `main` (BL-338).** Detection of no-HEAD / no-remote state, `--staged` / `--initial-scaffold` fallback modes. `/lp-commit` emits `Initial-Scaffold: true` trailer instead of the misleading `Mandatory-Review-Skipped: emergency-hotfix` value.
- **`compound-learning.sh` no longer hard-deps on `prd.json` (BL-335).** Consumes the canonical LaunchPad artifacts (PRD.md + scaffold-receipt.json); hand-authored `prd.json` remains an optional override.
- **Targeted kernel-file fallback (BL-341).** When kernel files are missing from a scaffolded project, `/lp-define` invokes `KernelRenderer.render_all(only_paths=missing)` so ONLY the absent files render — user-edited kernel files are not overwritten. A `SecretScannerViolation` or `OSError` aborts with an explicit `error:`; other failures log and continue.

## Highlights

- **BL-353 + BL-354 — TS-stack first-push CI green (P0).** `DEFAULT_PNPM_VERSION = "10.30.1"` + `DEFAULT_NODE_VERSION = "22.12.0"` constants in `plugin_stack_adapters/_constants.py`. `identity_inject` surfaces both as `default_pnpm_version` / `default_node_version` to every kernel/infrastructure render. `ci.yml.j2` declares `version: '{{ default_pnpm_version }}'` on `pnpm/action-setup`; new `nvmrc.j2` template renders `{{ default_node_version }}` to `.nvmrc` at repo root. `INFRASTRUCTURE_FILES` extends by 3 entries (count 31 → 34). Tests at `tests/test_ci_self_consistency_v215.py`.
- **BL-355 — workflow self-consistency assertion (P1).** `_validate_workflow_self_consistency(batch, cwd)` in `infrastructure_renderer.py` parses every rendered `.github/workflows/*.yml`, walks every step, and asserts that any `node-version-file` / `python-version-file` / `go-version-file` / `ruby-version-file` input names a file in the batch or on disk. `InfrastructureRenderer.render_all` raises `InfrastructureRenderError` on inconsistency. Defensive against malformed YAML — yaml.YAMLError skips the workflow rather than fails the check (test fixtures inject deliberate Jinja markup).
- **BL-333 — brainstorm consumer (P0).** `lp_define_runner.read_brainstorm_summary(repo_root) -> dict[str, str]` parses Markdown by `## ` heading, slug-normalizes (`Success Criteria` → `success_criteria`), applies alias map. `_build_jinja_context` injects as `brainstorm` key. PRD/APP_FLOW/BACKEND_STRUCTURE templates consume `brainstorm.*` when present. Defensive: missing file or malformed Markdown returns `{}` and renders the placeholder text unchanged.
- **BL-340 — build-runner non-TTY false-pass detection (P1).** `_run_cmd_with_prompt_detection(cmd, repo_root)` tees stdout/stderr to terminal AND scans line-by-line for `_PROMPT_BAIL_PATTERNS`. Returns `(returncode, prompt_pattern_or_None)`; caller treats exit-0 + pattern as failure.
- **BL-337 — `/lp-review` pre-first-commit fallback.** Step 1.A detection of `git rev-parse --verify HEAD` failure OR `git rev-parse --verify origin/main` failure. Fallback modes: `--staged` (review staged files) or working-tree default.
- **BL-338 — `/lp-commit` initial-scaffold mode.** Step 1.A auto-detection of no-HEAD state (or explicit `--initial-scaffold` flag). Skip Step 2.5 mandatory review (no diff base); allow commit on `main`; emit `Initial-Scaffold: true` trailer.
- **BL-341 — KernelRenderer fallback in `/lp-define`.** `_kernel_fallback_render(repo_root)` checks `KERNEL_FILES` against on-disk state; if any missing AND `scaffold-decision.json` provides an identity block, invokes `KernelRenderer.render_all(only_paths=missing)` so user-edited kernel files are never clobbered. `SecretScannerViolation` and `OSError` paths abort explicitly (NOT swallowed); other failures log a warning and let `/lp-define` continue.
