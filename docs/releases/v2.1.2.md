# LaunchPad v2.1.2

## v2.1.2 ship (this commit, post-PR-#65)

**v2.1.2 propagates the v2.1.1 self-host Python lefthook gates to consumer projects.** v2.1.1 hardened the plugin's own `lefthook.yml` with bandit + ruff + pyright + pytest; v2.1.2 carries those gates to projects scaffolded by `/lp-scaffold-stack` for the `nextjs_fastapi` stack via a shared template partial. The lane is intentionally narrow — only Python-bearing stacks get the new gates, and consumers who don't have the tools installed see a single fail-loud `GATE MISSING` line with the exact `pip install` command to fix it.

User-visible deltas over v2.1.1:

- **Consumer-side Python lefthook gates land on `nextjs_fastapi`.** Scaffolding a new `nextjs_fastapi` project produces a `lefthook.yml` that runs bandit + ruff-check + ruff-format-check on pre-commit and pyright + pytest on pre-push, scoped to the workspace's Python tree. Consumers without the tools installed get `GATE MISSING: <tool>. Install with: pip install '<tool>[>=minver]'.` — a single deterministic line per missing tool, no plugin-internal stack traces.
- **Pyright probes `apps/api/` first, then `api/`, then silent-skips.** Composition-layout projects (`apps/api/`) take precedence over the legacy single-stack layout (`api/`). If neither exists, the gate exits 0 silently — bare `pyright` cwd-walk is never invoked. Pytest tolerates exit 5 (no tests collected) so fresh scaffolds don't fail their first commit before any tests are written.
- **Per-stack scope: `nextjs_fastapi` only at v2.1.2.** Other v2.1 stacks (`ts_monorepo`, `nextjs_standalone`, `astro`, `generic`) have no Python workspace and inherit no Python gates. v2.2 stack adapters with Python workspaces drop in the new `_partials/_python_gates.j2.fragment` partial with one `{% include %}` line — no per-adapter duplication.
- **Tool minimum versions pinned in install hint**: `bandit>=1.7.10`, `ruff>=0.5`, `pyright>=1.1.350`, `pytest>=8`.

The lane also lands one new release-engineering invariant test (BL-319), one dead-code cleanup (BL-317), and one sibling-doc frontmatter sync (BL-321) as ride-along corrections.

## Migration from v2.1.1

No data migration required. v2.1.2 is purely additive on the consumer-side surface. Existing v2.1.1 scaffolds keep working unchanged — the new partial only fires when `/lp-scaffold-stack` is invoked against a `nextjs_fastapi` template.

The behavior change to be aware of: **consumer projects installing `nextjs_fastapi` will hit `GATE MISSING` on first commit until the propagated tools are installed.** The install hint is surfaced both in the gate failure message and in `/lp-bootstrap` output so it's discoverable at the natural moment. Recommended install:

```
pip install 'bandit>=1.7.10' 'ruff>=0.5' 'pyright>=1.1.350' 'pytest>=8'
```

Or per-tool, picking only the gates the project wants. To opt out of any gate, edit the generated `lefthook.yml` and delete the entry; the partial is rendered once at scaffold time, not on every run. For one-off skip across all gates, use the standard lefthook escape hatch: `LEFTHOOK=0 git commit ...`.

### Known limitations

- **Spaces-in-filenames in pre-commit gates.** The `lefthook.yml` `large-file-guard`, `trailing-whitespace`, and `end-of-file-newline` entries inherit the v2.1.1 maintainer pattern of double-quoting `{staged_files}` inside `<<< "$(echo ... )"`. A staged file whose pathname contains `$(...)` (POSIX-permitted but unusual) would cause the embedded substitution to evaluate. Defense-in-depth bug, not exploitable in normal use; tracked as **BL-313** for v2.1.4 with three documented remediation options (xargs -0 + NUL-delim, Python helper, or printf form).
- **`nodeenv` supply-chain caveat persists.** Pyright transitively pulls `nodeenv`, which downloads a Node.js binary at install time outside pip's hash protection. Tracked as **BL-299** for v2.1.4 remediation. Operators in regulated environments should pin/verify the node binary out-of-band.

## Upgrading

```
/plugin marketplace update builtform
/plugin uninstall launchpad --scope <same-scope>
/plugin install launchpad@builtform --scope <same-scope>
```

Restart Claude Code after updating. No migration step is required for the plugin itself — v2.1.1 → v2.1.2 is purely additive on the consumer-side scaffold surface.

For consumer projects using `nextjs_fastapi`, install the propagated tools after re-scaffolding (or before the first commit on a new scaffold):

```
pip install 'bandit>=1.7.10' 'ruff>=0.5' 'pyright>=1.1.350' 'pytest>=8'
```

Default `bandit -ll` posture means medium-or-higher severity findings fail the commit; lower-severity findings are advisory. If you want bandit to honor a `[tool.bandit]` block in `pyproject.toml`, edit the generated `lefthook.yml` bandit entry to `bandit -ll -c pyproject.toml -- "$@"` (the gate uses default config-file resolution otherwise).

For one-off skip across all gates: `LEFTHOOK=0 git commit -m "..."`.
