Consolidated work queue for the post-v0.1 hardening pass. v0.1 shipped on

## Deferred Codex findings (non-blocking)

### 1. Build-time secret scan â€” source-pattern superset (round 11, P1)

**Where:** `scripts/plugin-filter-secret-patterns.sh`, `scripts/build-plugin.sh:235`

**Issue:** The secret self-scan runs against the shipped filtered baseline
only. Source `.launchpad/secret-patterns.txt` is discarded before scanning,
which drops LaunchPad-internal patterns from the build-time safety net.

**Why deferred:** Current risk is low â€” all high-signal format-specific
patterns (AWS, GitHub, Stripe, Slack, OpenAI/Anthropic, private-key headers)
are in the shipped baseline and catch the 95% case. The dropped source
patterns are overly broad (`password=`, `token=`, `Bearer <token>`,
URL-with-creds) and would false-positive on plugin docs/prompts if applied
verbatim.

**Proper fix (not "add source patterns to scan"):**

1. Keep shipping the filtered baseline (correct â€” internal patterns stay private)
2. Maintain a **separate `.launchpad/secret-patterns-scan-only.txt`** with
   high-signal internal patterns converted to POSIX ERE syntax
3. Union `scan-only + filtered baseline` at build time for self-scan
4. Shipped `plugin/data/secret-patterns.txt` continues to hold only the
   filtered baseline

**Estimated effort:** ~30 min (new file + build script wiring)

### 2. Dynamic harness command collection (round 11, P2)

**Where:** `scripts/plugin-command-rewrites.py:30`

**Issue:** `HARNESS_COMMANDS = {"build", "plan", "define", "kickoff"}` is
hardcoded. A new `.claude/commands/harness/foo.md` added later would ship
to plugin but references to `/harness:foo` elsewhere wouldn't rewrite to
`/lp-harness-foo`.

**Why deferred:** Zero runtime impact on v0.1 (all 4 current harness
commands are in the hardcoded set). Only bites when new harness commands
are added.

**Proper fix:**

1. In `build_rules()`, collect harness command names from
   `src/harness/*.md` dynamically (same pattern as top-level `cmds`).
2. Add an assertion to `plugin-verify-rewrites.sh` for surviving
   `/harness:<name>` references.

**Estimated effort:** ~15 min

## Rule-based CI scanners (new recommendation from review loop)

### 7. shellcheck integration

11 rounds of Codex review surfaced mechanical shell bugs (regex gaps,
operator handling, word-boundary issues) one at a time. `shellcheck` as a
CI step would catch most of these deterministically in a single pass.

**Integration:** add to `.github/workflows/ci.yml` with allowlist for any
intentional exceptions.

### 8. gitleaks integration

Complementary to the plugin-build secret self-scan. `gitleaks` runs across
the entire git history and catches known token formats across a broader
corpus than our hand-curated baseline.

**Integration:** add to `.github/workflows/ci.yml`, configured to fail on
high-severity finds.
